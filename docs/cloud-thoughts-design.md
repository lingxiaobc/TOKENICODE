# 「云随想」完整功能设计方案

## Context

TOKENICODE 目前是一个桌面端 Claude Code GUI，用户量已达数百人。产品方向是将 MemoryWork 能力融入 TOKENICODE，让用户打开即有一套成型的记忆工作工具。

「云随想」是这个方向的 MVP 第一步——一个跨端的想法收集与同步模块。用户在移动端或桌面端随时记录灵感、语音、文件，所有内容云端同步，桌面端下次对话时可拉取使用。

---

## 一、产品定义

### 云随想是什么

一个轻量的跨端想法收集器。核心价值：**随时随地捕捉想法，回到桌面端时一切就绪。**

### Local-First 设计原则

云随想采用**本地优先**架构——所有核心功能不依赖服务器，离线即可使用。云同步是可选的增强能力。

```
Layer 0 · 本地模式（零配置，开箱即用）
├── 桌面端创建/编辑/删除/搜索随想
├── 引用到对话 / 保存对话为随想
├── 语音录制 + 实时转写（火山引擎 STT）
└── 数据存储在本地（SQLite）

Layer 1 · 云同步模式（配对后解锁）
├── 移动端 ↔ 桌面端双向同步
└── 云端备份
```

用户装完 TOKENICODE 就能直接使用云随想的本地功能，不需要注册、配对或联网。想跨端同步时再配对。

### 核心功能（移动端和桌面端一致）

| 功能 | 说明 |
|------|------|
| **文字记录** | 快速输入文字，支持简单 Markdown |
| **语音便签** | 录音时实时 STT 转写，保存语音+文字双份内容 |
| **小文件上传** | 图片、截图、代码片段、PDF（≤10MB/文件） |
| **云同步** | 新增/编辑/删除实时同步到所有端 |
| **搜索与标签** | 按关键词搜索、标签过滤 |

### 双向打通能力（桌面端）

| 功能 | 方向 | 说明 |
|------|------|------|
| **引用到对话** | 随想→对话 | 将随想条目作为上下文素材注入 Claude Code 对话 |
| **拖拽注入** | 随想→对话 | 拖拽随想卡片到 InputBar 输入框 |
| **保存为随想** | 对话→随想 | 对话中的消息右键「保存为随想」，同步到移动端 |
| **新消息提醒** | 双向 | 有新的移动端随想时，桌面端显示未读徽章 |

## 二、已确认的决策

| 决策项 | 选择 | 说明 |
|--------|------|------|
| **移动端形态** | 独立 APP | 使用 React Native (Expo) 开发，iOS + Android 双端 |
| **与对话关系** | 双向打通 | 随想可注入对话 + 对话中的内容也能保存为随想 |
| **用户身份** | 配对码 | 桌面端生成 6 位码，移动端输入绑定，零注册 |
| **语音转写** | 火山引擎大模型 ASR | WebSocket 流式，录音时实时转写，桌面端+移动端统一方案 |

### 为什么选 React Native (Expo)

- TOKENICODE 前端是 React + TypeScript，**技术栈完全一致**，可复用类型定义和部分逻辑
- Expo 简化了原生能力接入（录音、文件选择、推送通知），不需要碰 Xcode/Android Studio
- 可以通过 Expo Go 快速测试，不需要每次都构建原生包
- 上架 App Store / Google Play / 国内安卓市场

### 双向打通的具体含义

```
方向 1：随想 → 对话
- 云随想面板中点击「引用」或拖拽 → 随想内容注入到 InputBar
- Claude Code 可以读取随想内容作为上下文

方向 2：对话 → 随想
- 对话中的 AI 回复或用户消息，右键菜单「保存为随想」
- 保存时可选择添加标签、编辑备注
- 保存后在云随想面板和移动端 APP 中可见
```

## 三、数据模型

### Thought（随想条目）

```typescript
interface Thought {
  id: string;                    // UUID
  user_id: string;               // 配对 token（关联桌面端和移动端）

  // 内容
  content: string;               // 文字内容（Markdown）
  type: 'text' | 'voice' | 'file' | 'mixed';  // 条目类型

  // 语音便签
  voice_url?: string;            // 语音文件 COS 路径
  voice_duration?: number;       // 语音时长（秒）
  voice_transcript?: string;     // 火山引擎 STT 实时转写文字（录音时同步生成）

  // 附件
  attachments?: Attachment[];

  // 元数据
  tags?: string[];               // 用户标签
  source: 'mobile' | 'desktop';  // 来源设备
  is_read: boolean;              // 桌面端是否已读

  // 时间
  created_at: number;            // Unix timestamp
  updated_at: number;
  deleted_at?: number;           // 软删除
}

interface Attachment {
  id: string;
  filename: string;
  file_url: string;              // COS 路径
  file_size: number;             // 字节
  mime_type: string;
  thumbnail_url?: string;        // 图片缩略图
}
```

### UserPair（设备配对）

```typescript
interface UserPair {
  id: string;                    // 配对 token
  pair_code: string;             // 6 位配对码（临时，配对后失效）
  desktop_device_id: string;     // 桌面端设备标识
  mobile_device_id?: string;     // 移动端设备标识（配对后写入）

  created_at: number;
  paired_at?: number;            // 配对成功时间
  last_sync_at?: number;         // 最后同步时间
}
```

```
├── volcengine-stt.ts     ← 火山引擎 ASR WebSocket 客户端（双端共享）
├── sync-logic.ts         ← 同步队列、冲突解决逻辑
└── validators.ts         ← 输入校验规则
```

### 后端 API 设计

```
POST   /api/pair/generate        ← 桌面端生成配对码
POST   /api/pair/bind            ← 移动端输入配对码绑定
GET    /api/pair/status           ← 检查配对状态

GET    /api/thoughts              ← 获取随想列表（分页、筛选）
POST   /api/thoughts              ← 创建随想
PUT    /api/thoughts/:id          ← 更新随想
DELETE /api/thoughts/:id          ← 删除随想（软删除）
GET    /api/thoughts/unread       ← 获取未读数量

POST   /api/upload/presign        ← 获取 COS 预签名上传 URL
```

### 认证机制

```
配对流程：
1. 桌面端请求 POST /api/pair/generate → 返回 { pair_code: "482916", token: "xxx" }
2. 桌面端显示 6 位配对码，用户在手机上输入
3. 移动端请求 POST /api/pair/bind { pair_code: "482916" } → 返回 { token: "xxx" }
4. 双端都持有 token，后续请求在 Header 中带 Authorization: Bearer <token>

安全措施：
- 配对码 5 分钟过期
- 3 次错误输入后 15 分钟冷却
- Token 有效期 90 天，可刷新
- 所有 API 走 HTTPS
```

## 五、移动端设计（React Native APP）

### 技术栈

| 组件 | 方案 | 说明 |
|------|------|------|
| 框架 | **Expo (React Native)** | 与桌面端 React+TS 技术栈一致，可复用类型定义 |
| UI 库 | React Native Paper 或 NativeWind (Tailwind) | NativeWind 可复用桌面端 Tailwind 样式经验 |
| 语音录制 | `expo-av` | 支持录音 + 播放 |
| 语音转写 STT | 火山引擎大模型流式语音识别 | WebSocket 实时转写，支持中英混说+方言 |
| 文件选择 | `expo-document-picker` + `expo-image-picker` | 图片、文件选择 |
| 文件上传 | COS SDK + 预签名直传 | 直传 COS，不经后端中转 |
| 网络请求 | axios + token 拦截器 | 与桌面端 cloud-api.ts 可共享逻辑 |
| 本地存储 | `expo-secure-store` + AsyncStorage | Token 安全存储 + 离线缓存 |
| 推送通知 | `expo-notifications` | 桌面端有新随想时推送（可选） |

### 方向 1：随想 → 对话（引用）

两种方式：
1. **点击「引用」按钮** → 随想内容以引用块格式插入 InputBar
2. **拖拽卡片** → 拖到 ChatPanel 的 InputBar 区域松手

注入格式：
```
[云随想引用]
---
{随想文字内容}
{如有附件: [附件名称](本地缓存路径)}
{如有语音转写: 语音转写: "xxx"}
---
```

### 方向 2：对话 → 随想（保存）

在 **MessageBubble** 组件的右键菜单（或 hover 操作栏）新增「保存为随想」选项：

```
交互流程：
1. 用户在对话中 hover 某条消息 → 出现操作栏
2. 点击「保存为随想」图标
3. 弹出迷你编辑框：
   - 自动填充消息内容摘要
   - 可添加标签、编辑备注
   - 标记来源为「对话摘要」
4. 点击保存 → 创建随想条目 → 同步到云端 → 移动端可见
```

需修改的现有组件：
- `src/components/chat/MessageBubble.tsx` — 新增操作按钮
- 或在现有的 hover 操作栏中加一个「保存」图标

## 七、同步机制

### 同步策略

```
拉取策略（桌面端）：
1. 应用启动时拉取最新随想列表
2. 每 30 秒轮询新消息（GET /api/thoughts/unread）
3. 手动刷新按钮

推送策略（双端）：
1. 创建/编辑/删除时立即调 API
2. 失败时存入本地队列，下次联网重试

冲突处理（MVP 简化）：
- 随想条目不支持多端同时编辑同一条
- 以最后写入时间戳为准（Last Write Wins）
- 删除操作优先于编辑操作
```

### 离线支持

```
移动端（React Native APP）：
- 新建随想时先存 AsyncStorage
- 联网后自动同步到云端
- 离线草稿标记「待同步」状态

桌面端（TOKENICODE）：
- 随想列表缓存在本地（thoughtsStore 持久化）
- 新建/编辑操作先写本地再推云端
- 网络恢复后自动重试队列
```

## 八、语音便签技术方案（火山引擎大模型 ASR）

### 工作流程

```
用户按下录音 → expo-av 开始录音（16kHz/16bit）
      │
      ├─→ 音频流通过 WebSocket 实时发送到火山引擎 ASR
      │         │
      │         └─→ 实时返回转写文字，UI 上同步显示
      │
用户松手停止录音
      │
      ├─→ 保存原始语音文件（上传 COS）
      ├─→ 保存最终转写文字（写入 content / voice_transcript）
      └─→ 创建 type: 'voice' 的随想条目
```

### 接入参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 协议 | WebSocket | 双端统一 |
| 识别模式 | `bigmodel_nostream` | 流式输入，录完出结果，准确度最高 |
| 采样率 | 16000 Hz | expo-av 录音配置 |
| 位深 | 16 bit | 单声道 |
| 单包大小 | 200ms | 性能最优 |
| `enable_ddc` | true | 语义顺滑，去掉停顿词（嗯、啊、那个） |
| `enable_punc` | true | 自动标点 |

### 鉴权

```
鉴权走后端代理，不在客户端暴露 Key：
1. 客户端请求后端获取临时 WebSocket 连接信息
2. 后端签发带时效的连接参数
3. 客户端直连火山引擎 WebSocket（音频流不经后端中转）

Header：
- X-Api-App-Key: APP ID
- X-Api-Access-Key: 访问令牌
- X-Api-Resource-Id: 资源类型
- X-Api-Connect-Id: UUID（追踪用）
```

### 支持语言

默认空值即支持：中文、英文、中英混说、上海话、闽南语、四川话、陕西话、粤语。无需用户手动选择。

### 桌面端接入

桌面端通过 Tauri Rust 后端或前端 WebSocket 直连火山引擎 ASR，复用同一套鉴权和协议逻辑。

### 共享代码

```
shared/
├── volcengine-stt.ts         ← WebSocket 连接、二进制协议封装、转写回调
└── types.ts                  ← STT 相关类型定义
```

移动端和桌面端共享 `volcengine-stt.ts`，录音组件各自实现（expo-av / Web Audio API）。

## 十一、风险与注意事项

1. **App Store 审核**：iOS 审核对「笔记类 + 需要配合桌面端使用」的 APP 可能有疑问，需要准备清晰的 App 描述和演示视频
2. **国内安卓分发**：各市场（华为、小米、应用宝等）需要软件著作权，申请周期 1~2 个月。MVP 阶段可先走 APK 直接下载 + Google Play
3. **语音文件大小**：60 秒语音约 1MB，需要设置录制时长上限（建议 5 分钟）
4. **COS 流量费**：如果用户频繁下载语音/文件，流量费可能超预期。可设置文件本地缓存策略
5. **配对码安全**：6 位数字码空间较小（100 万种），需要防暴力破解（限速 + 冷却）
6. **数据隐私**：用户随想可能包含敏感内容，服务端存储需加密，隐私政策需明确
7. **Expo 限制**：部分原生能力（如后台录音、蓝牙等）可能需要 bare workflow 而非 managed workflow。语音录制和文件上传在 managed workflow 下可行
